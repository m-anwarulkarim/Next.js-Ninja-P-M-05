# Next.js Image Component - সম্পূর্ণ ব্যাখ্যা

Next.js এর `Image` component হলো একটি powerful built-in component যা আপনার website এর images automatically optimize করে দেয়। চলুন step by step বুঝি:

## কেন এটি ব্যবহার করবেন?

সাধারণ HTML `<img>` tag এর তুলনায় Next.js Image component অনেক সুবিধা দেয়:

**১. Automatic Optimization:** Images automatically compress এবং modern format (WebP, AVIF) এ convert হয়, যা load time কমায়।

**২. Lazy Loading:** শুধুমাত্র viewport এ আসলে image load হয়, পুরো page একসাথে load হয় না।

**৩. Responsive:** Different screen size এর জন্য automatically সঠিক size এর image serve করে।

**৪. No Layout Shift:** Image load হওয়ার সময় page layout jump করে না (CLS problem solve)।

## Basic ব্যবহার

```javascript
import Image from "next/image";

export default function Profile() {
  return (
    <Image src="/profile.jpg" width={500} height={500} alt="Profile Picture" />
  );
}
```

## গুরুত্বপূর্ণ Props

**src:** Image এর path বা URL। Local file হলে `/public` folder থেকে শুরু হবে।

**width ও height:** Image এর dimension দিতে হবে। এটা দিলে browser জায়গা reserve করে রাখে, ফলে layout shift হয় না।

**alt:** Image এর description, accessibility এর জন্য জরুরি।

**priority:** Hero image বা above-the-fold image এর জন্য `priority={true}` দিলে সেটা lazy load না হয়ে immediately load হবে।

**fill:** Parent container এর সাথে image fit করতে চাইলে width/height এর পরিবর্তে `fill` ব্যবহার করুন।

```javascript
<div style={{ position: "relative", width: "100%", height: "400px" }}>
  <Image src="/banner.jpg" fill style={{ objectFit: "cover" }} alt="Banner" />
</div>
```

## External Images

বাইরের website থেকে image load করতে চাইলে `next.config.js` এ domains configure করতে হবে:

```javascript
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "example.com",
      },
    ],
  },
};
```

## Responsive Images

Different breakpoints এর জন্য different sizes specify করতে পারেন:

```javascript
<Image
  src="/photo.jpg"
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
  fill
  alt="Responsive photo"
/>
```

এই configuration বলছে: mobile এ full width (100vw), tablet এ half width (50vw), desktop এ এক-তৃতীয়াংশ width (33vw)।

## সাধারণ ভুল এবং সমাধান

**ভুল:** Width/height না দেওয়া (fill ছাড়া)
**সমাধান:** সবসময় width ও height specify করুন অথবা fill property ব্যবহার করুন।

**ভুল:** External URL এর জন্য domain configure না করা
**সমাধান:** next.config.js এ remotePatterns add করুন।

**ভুল:** Parent container এ position: relative না থাকা (যখন fill ব্যবহার করছেন)
**সমাধান:** Parent element এ `position: relative` দিন।

এই component সঠিকভাবে ব্যবহার করলে আপনার website এর performance অনেক improve হবে এবং Google এর Core Web Vitals score ভালো আসবে।
