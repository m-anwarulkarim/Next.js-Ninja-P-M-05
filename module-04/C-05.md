Web development-এ **SSG, SSR, ISR,** এবং **CSR** হলো ডেটা রেন্ডারিং করার ৪টি প্রধান পদ্ধতি। সহজ কথায় বলতে গেলে, এগুলো ঠিক করে যে আপনার ওয়েবসাইট বা ওয়েব পেইজটি ইউজারের সামনে কীভাবে তৈরি হয়ে আসবে।

নিচে একদম সহজ ভাষায় এগুলোর পার্থক্য বুঝিয়ে দিচ্ছি:

---

## ১. CSR (Client-Side Rendering)

**মূল কনসেপ্ট:** সার্ভার থেকে শুধু একটা ফাঁকা ফাইল (Skeleton) আসে। এরপর ব্রাউজারে থাকা JavaScript ফাইলটি রান হয়ে পুরো পেইজটি তৈরি করে।

- **কীভাবে কাজ করে:** ইউজার লিংকে ক্লিক করলে ব্রাউজার সার্ভার থেকে ডাটা নিয়ে আসে এবং নিজের ক্ষমতা ব্যবহার করে পেইজটি সাজায়।
- **সুবিধা:** একবার লোড হয়ে গেলে অন্য পেইজে যাওয়া খুব দ্রুত হয় (যেমন: Facebook বা Gmail)।
- **অসুবিধা:** প্রথমবার লোড হতে সময় নেয়। তাছাড়া SEO (Search Engine Optimization)-এর জন্য এটি খুব একটা ভালো নয়।

এখানে ডাটা ফেচ করার জন্য আমরা `useEffect` ব্যবহার করি। পেইজটি প্রথমে ব্রাউজারে লোড হয়, তারপর ডাটা আসে।

```javascript
"use client"; // ক্লায়েন্ট সাইড রেন্ডারিংয়ের জন্য এটি জরুরি

import { useState, useEffect } from "react";

export default function CSRPage() {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch("https://api.example.com/products")
      .then((res) => res.json())
      .then((data) => setData(data));
  }, []);

  if (!data) return <p>Loading...</p>;

  return <div>{data.name}</div>;
}
```

## ২. SSG (Static Site Generation)

**মূল কনসেপ্ট:** ওয়েবসাইট বানানোর সময় (Build Time) সব পেইজ আগেই তৈরি করে রাখা হয়।

- **কীভাবে কাজ করে:** আপনি যখন কোড পুশ করেন বা সাইটটি তৈরি করেন, তখনই সব HTML পেইজ জেনারেট হয়ে যায়। ইউজার রিকোয়েস্ট করার সাথে সাথে সার্ভার থেকে তৈরি করা ফাইলটি চলে আসে।
- **সুবিধা:** এটি সবচাইতে দ্রুত লোড হয় এবং সিকিউরিটি অনেক ভালো। ব্লগের জন্য এটি সেরা।
- **অসুবিধা:** ডেটা আপডেট করতে হলে আপনাকে আবার পুরো সাইট "Build" করতে হবে। রিয়েল-টাইম ডেটার জন্য এটি ভালো নয়।
  Next.js-এ ডিফল্টভাবে সব কম্পোনেন্ট স্ট্যাটিক থাকে। যদি আপনি কোনো এক্সট্রা কনফিগারেশন না করেন, তবে এটি SSG হিসেবে কাজ করবে।

```javascript
// এটি ডিফল্টভাবেই SSG
async function getStaticData() {
  const res = await fetch("https://api.example.com/about-us");
  return res.json();
}

export default async function SSGPage() {
  const data = await getStaticData();

  return (
    <div>
      <h1>{data.title}</h1>
      <p>এটি বিল্ড টাইমে তৈরি হয়ে থাকবে।</p>
    </div>
  );
}
```

## ৩. SSR (Server-Side Rendering)

**মূল কনসেপ্ট:** ইউজার যখনই রিকোয়েস্ট করে, সার্ভার তখনই সাথে সাথে পেইজটি তৈরি করে ব্রাউজারে পাঠায়।

- **কীভাবে কাজ করে:** ইউজার ক্লিক করলে সার্ভার ডেটাবেস থেকে তথ্য নেয়, পেইজটি সাজায় এবং রেডিমেড HTML ফাইল ব্রাউজারে পাঠায়।
- **সুবিধা:** SEO-এর জন্য এটি সেরা এবং ইউজার সবসময় লেটেস্ট বা টাটকা ডেটা দেখতে পায় (যেমন: ই-কমার্স বা নিউজ সাইট)।
- **অসুবিধা:** সার্ভারের ওপর অনেক চাপ পড়ে এবং প্রতিবার রিকোয়েস্টে সার্ভার কাজ করে বলে কিছুটা সময় নিতে পারে।

সার্ভার সাইড রেন্ডারিং করার জন্য আমাদের `fetch` রিকোয়েস্টে `cache: 'no-store'` ব্যবহার করতে হয়। এতে প্রতিবার রিকোয়েস্টে নতুন ডাটা আসে।

```javascript
async function DynamicData() {
  // no-store মানে হলো ডাটা ক্যাশ হবে না, প্রতিবার নতুন আসবে (SSR)
  const res = await fetch("https://api.example.com/stock-price", {
    cache: "no-store",
  });
  return res.json();
}

export default async function SSRPage() {
  const data = await DynamicData();

  return (
    <div>
      <h1>ডলার রেট: {data.rate}</h1>
      <p>এটি প্রতিবার রিকোয়েস্ট করলে সার্ভারে তৈরি হবে।</p>
    </div>
  );
}
```

## ৪. ISR (Incremental Static Regeneration)

**মূল কনসেপ্ট:** এটি SSG এবং SSR এর একটি হাইব্রিড ভার্সন। পুরো সাইট নতুন করে বিল্ড না করেই নির্দিষ্ট সময় পরপর নির্দিষ্ট পেইজ আপডেট করা যায়।

- **কীভাবে কাজ করে:** আপনি একটি সময় সেট করে দিলেন (যেমন: ৬০ সেকেন্ড)। ৬০ সেকেন্ড পর কোনো ইউজার এলে সাইটটি ব্যাকগ্রাউন্ডে অটোমেটিক ডেটা আপডেট করে নেবে।
- **সুবিধা:** SSG-এর মতো ফাস্ট স্পিড পাওয়া যায়, আবার SSR-এর মতো লেটেস্ট ডেটাও পাওয়া যায়।
- **অসুবিধা:** এটি সেটআপ করা কিছুটা জটিল।

ISR করার জন্য `fetch`-এর ভেতরে `next: { revalidate: 60 }` দিতে হয়। এর মানে হলো প্রতি ৬০ সেকেন্ড পর পর ডাটা আপডেট হবে।

```javascript
async function getRevalidatedData() {
  // ৬০ সেকেন্ড পর পর ব্যাকগ্রাউন্ডে ডাটা আপডেট হবে
  const res = await fetch("https://api.example.com/blog-posts", {
    next: { revalidate: 60 },
  });
  return res.json();
}

export default async function ISRPage() {
  const data = await getRevalidatedData();

  return (
    <div>
      <h1>ব্লগ পোস্ট</h1>
      {/* ডাটা এখানে রেন্ডার হবে */}
    </div>
  );
}
```

### একনজরে পার্থক্য (Comparison Table)

| ফিচার               | CSR                       | SSG                        | SSR                           | ISR                                |
| ------------------- | ------------------------- | -------------------------- | ----------------------------- | ---------------------------------- |
| **কখন রেন্ডার হয়?** | ব্রাউজারে (ইউজার আসার পর) | বিল্ড টাইমে (আগে থেকে)     | সার্ভারে (প্রতিটি রিকোয়েস্টে) | ব্যাকগ্রাউন্ডে (নির্দিষ্ট বিরতিতে) |
| **স্পিড**           | প্রথমবার স্লো, পরে ফাস্ট  | সবচেয়ে বেশি ফাস্ট          | মিডিয়াম                       | অনেক ফাস্ট                         |
| **SEO**             | খারাপ                     | খুব ভালো                   | খুব ভালো                      | খুব ভালো                           |
| **ডেটা আপডেট**      | রিয়েল-টাইম                | কোড নতুন করে বিল্ড করতে হয় | রিয়েল-টাইম                    | নির্দিষ্ট সময় পরপর আপডেট হয়        |

---

**সহজ উদাহরণ:**

- **CSR:** একটা খালি থালা দেওয়া হলো, আপনি নিজের মতো খাবার সাজিয়ে নিলেন।
- **SSG:** প্যাকেট করা খাবার (আগে থেকেই রেডি)।
- **SSR:** অর্ডার দেওয়ার পর রেস্টুরেন্টে রান্না করে দেওয়া ফ্রেশ খাবার।
- **ISR:** কিছুক্ষণ পরপর নতুন খাবার দিয়ে স্টল আপডেট করা।
