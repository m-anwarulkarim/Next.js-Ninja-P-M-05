# public Folder

Next.js প্রজেক্টের রুট ডিরেক্টরিতে (Root Directory) থাকা **`public`** ফোল্ডারটি হলো এমন একটি জায়গা যেখানে আপনি আপনার অ্যাপ্লিকেশনের **Static Assets** বা স্থির ফাইলগুলো রাখেন। এটি মূলত সরাসরি ওয়েব থেকে অ্যাক্সেসযোগ্য একটি স্টোরেজ।

নিচে বিস্তারিত আলোচনা করা হলো:

---

### ১. `public` ফোল্ডার আসলে কী?

এটি একটি স্পেশাল ফোল্ডার যার ভেতরে রাখা ফাইলগুলো Next.js কোনো প্রকার প্রসেসিং বা বান্ডলিং (Bundling) ছাড়াই সরাসরি ব্রাউজারে সার্ভ করে। আপনি এই ফোল্ডারের ফাইলগুলোকে সরাসরি রুট URL ( `/` ) দিয়ে অ্যাক্সেস করতে পারেন।

---

### ২. কেন এটি ব্যবহার করবেন? (The "Why")

সব ফাইলকে আপনি কোডের ভেতরে বা `src` ফোল্ডারে রাখতে পারেন না। `public` ফোল্ডার দরকার হয় কারণ:

- **Static Assets:** লোগো, ইমেজ, আইকন, ফন্ট বা ভিডিও রাখার জন্য।
- **Metadata Files:** যেমন `favicon.ico`, `robots.txt`, বা `sitemap.xml` যা সার্চ ইঞ্জিন বা ব্রাউজারের জন্য সরাসরি প্রয়োজন হয়।
- **Downloaded Files:** যদি আপনি চান ইউজার আপনার সাইট থেকে কোনো PDF বা ফাইল ডাউনলোড করুক।
- **Direct Access:** কোনো স্ক্রিপ্ট বা ফাইল যা কোনো রকম ইমপোর্ট (Import) ছাড়াই সরাসরি URL দিয়ে দেখা দরকার।

---

### ৩. কিভাবে ব্যবহার করবেন? (The "How")

`public` ফোল্ডারের সব ফাইল আপনার অ্যাপ্লিকেশনের মূল ডোমেইনের ঠিক পরেই থাকে। আপনাকে পাথের শুরুতে `public` লিখতে হয় না।

**উদাহরণ ফোল্ডার স্ট্রাকচার:**

```text
my-app/
├── public/
│   ├── logo.png
│   ├── data/
│   │   └── manual.pdf
│   └── robots.txt
├── app/
│   └── page.tsx

```

#### ইমেজের ক্ষেত্রে ব্যবহার:

```tsx
import Image from "next/image";

export default function Page() {
  return (
    <div>
      {/* লক্ষ্য করুন: পাথে /public/logo.png লিখতে হয় না, শুধু /logo.png */}
      <img src="/logo.png" alt="Logo" />

      {/* Next.js Image Component এর মাধ্যমে */}
      <Image src="/logo.png" alt="Logo" width={100} height={100} />
    </div>
  );
}
```

#### সরাসরি ফাইল অ্যাক্সেস:

আপনি যদি ব্রাউজারে `localhost:3000/robots.txt` লিখে সার্চ দেন, তবে সরাসরি ওই ফাইলটি দেখতে পাবেন।

---

### ৪. গুরুত্বপূর্ণ নিয়ম ও সতর্কতা ⚠️

- **No `public` in path:** ফাইলে এক্সেস করার সময় কখনো `/public/image.png` লিখবেন না। এটি ভুল। সঠিক হলো `/image.png`।
- **Filename Conflict:** আপনার `public` ফোল্ডারে যদি `hello.html` নামে একটি ফাইল থাকে এবং আপনার `app/` ফোল্ডারেও যদি `hello/page.tsx` নামে একটি রাউট থাকে, তবে নেক্সট জেএস ফাইলটিকে (public) প্রায়োরিটি দিবে।
- **Security:** মনে রাখবেন, `public` ফোল্ডারে রাখা প্রতিটি ফাইল ইন্টারনেটে যে কেউ সরাসরি URL দিয়ে দেখতে পাবে। তাই এখানে কখনো সিক্রেট কোনো ফাইল বা এনভায়রনমেন্ট ভেরিয়েবল রাখবেন না।
- **Static Assets only:** এখানে আপনি কোনো `.tsx` বা `.js` ফাইল রাখলে সেটি কোড হিসেবে রান করবে না, বরং টেক্সট ফাইল হিসেবে ব্রাউজারে ডাউনলোড বা শো হবে।

---

### ৫. কখন `public` ব্যবহার করবেন আর কখন `src/assets`?

| বৈশিষ্ট্য            | `public` ফোল্ডার                 | `src/assets` (বা অন্য ফোল্ডার)        |
| -------------------- | -------------------------------- | ------------------------------------- |
| **অ্যাক্সেস পদ্ধতি** | সরাসরি URL দিয়ে (`/img.png`)     | ইমপোর্ট করে (`import img from '...'`) |
| **অপ্টিমাইজেশন**     | নেক্সট জেএস অটো-অপ্টিমাইজ করে না | নেক্সট জেএস ফাইলটি প্রসেস ও হ্যাশ করে |
| **ব্যবহার**          | Favicon, Robots, বড় ভিডিও, PDF   | ছোট আইকন, লোগো, CSS এর ভেতর ইমেজ      |
